# Evolith Ontology v1.0

This represents the frozen v1.0 schema for knowledge extraction and canonical representation.

## Nodes
- **Type**: `Technology`, `Concept`, `Repository`, `Paper`, `Package`, `Problem`, `RFC`
- **Knowledge DNA**: 
  - `memory_model` (e.g. Garbage Collected, Manual, Borrow Checker)
  - `type_system` (e.g. Static, Dynamic, Gradual, Duck)
  - `paradigms` (e.g. Object-Oriented, Functional, Reactive)
  - `target_domains` (e.g. Web, Systems, Embedded, Data Science)

## Edges (Relationships)
- `EVOLVES_INTO`
- `INFLUENCES`
- `OBSOLETES`
- `IMPLEMENTS`
- `USES`

## Mutations
When a technology evolves into another (`EVOLVES_INTO` or `INFLUENCES`), mutations must be tracked:
- `features_added`: list of new architectural features
- `features_removed`: list of dropped features

*Note: All future changes to this ontology must be introduced via formal versioning (e.g., v1.1, v2.0) with accompanying migration scripts.*
